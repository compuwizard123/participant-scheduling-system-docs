\section{Executive Summary}
This document's purpose is to detail the participant scheduling system proposed by the \index{Human-Computer Interaction Lab} Human-Computer Interaction Lab of Wisconsin-Madison. It is the fourth document describing this project, and enumerates test cases for the use cases presented in Milestone 2, as well as covering coding guidelines and change control.  The project exists because the lab wishes to unify their schedule information and provide a simple, intuitive interface for prospective participants to sign up for experiments.
